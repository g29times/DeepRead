# DEEPREAD
你的智能阅读助手

## 核心设计
将页面分为主页和工作区
主页：原页面内容，可以选任意文本，进行AI上下文解释
工作区：分为概念解释区（上半）和对话区（下半）
    概念解释区支持：全文概括，关键段落概括，关键词解释，关键句解释这四种核心场景
    对话区支持：普通文本对话，带图片的附件理解，google原生的url阅读能力和search搜索能力

主文件 content.js.

  - 金句热力 UI 的单页静态 mock（用于快速迭代交互与视觉）

## 远期设计
“动态学习用户划线风格”怎么接在后面（先埋口，不现在做）
第二阶段可以在请求里加：

userStyleProfile（来自历史行为统计：长度分布、是否偏好比喻句/定义句/结论句、偏好哪些词性或信号词等）
或更简单：把最近 N 条用户锚点的文本作为 few-shot，让模型“模仿用户的划线偏好”
触发条件就按你说的阈值：

文章足够长/段落足够多
用户标记 >= N（比如 8 或 12）
这块确实可以后置，不影响第一阶段。
